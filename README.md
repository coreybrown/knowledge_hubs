# knowledge_hubs

Place for deep learning — a collection of interlinked, in-depth knowledge bases.

## Knowledge Bases

### [IPAs](./IPAs)
A 154-note interlinked Obsidian knowledge base on the **India Pale Ale**, spanning
its contested 18th-century origins through the modern hazy/thiol-driven frontier.
Covers history, styles, ingredients, brewing, science & sensory, drinking, best-of
lists, industry & culture, and reference — connected by 3,163 verified wikilinks.

*Added 2026-05-21.*

### [Pour Over Coffee](./Pour%20Over%20Coffee)
A 144-note interlinked Obsidian knowledge base on **pour over coffee** — manual
filter brewing — with immersion methods (AeroPress, Clever, French press, OXO Rapid
Brewer) covered as adjacent techniques. Spans history & origins, equipment & drippers,
beans & roast, grinding, water, brewing technique, a deep recipes domain (26 official,
competition, and community recipes with machine-readable specs), science & extraction,
tasting & sensory, culture & industry, and reference — connected by 2,859 verified
wikilinks. Ships with a generated static website in `site/`.

*Added 2026-06-02.*

## Live site

All hubs are published together through the **Field Guides** portal —
<https://field-guides.vercel.app> (hubs at `/pour-over/` and `/ipas/`). The portal is
generated by `build_landing.py`, which renders the landing page and assembles each hub's
built `site/` into `docs/` under a clean slug. Vercel builds `docs/` on every push to
`main` (`docs/` is git-ignored, built on deploy).

## Ask the Guide (AI assistant)

Each hub's search box (`/`) also answers natural-language questions, grounded **only** in
that guide's own notes, with "Go deeper" links to the source pages. A zero-dependency
Vercel function (`api/ask.js`) retrieves the most relevant notes from the guide's
`ask-corpus.json` (emitted at build time) and streams a concise, cited answer from
**Claude Sonnet 4.6**. It's per-IP and globally rate-limited (Upstash KV) under a fixed
monthly spend cap. Design, model, and limits live in
[`ASK_FEATURE_SPEC.md`](./ASK_FEATURE_SPEC.md).

> **`api/` functions must be plain CommonJS `.js`, not TypeScript** — this project's
> Python-first Vercel build compiles `.ts` to ESM but runs it as CommonJS, which crashes
> every request with `FUNCTION_INVOCATION_FAILED`.

## Tooling

### `verify_kb.py`
Consistency checker for any knowledge base in this repo. Validates that every note
matches the `Note Registry`, every `[[wikilink]]` resolves, no link spans a line
break, and no note is orphaned.

```
python3 verify_kb.py <KB folder>        # report
python3 verify_kb.py <KB folder> --fix  # also reflow multi-line wikilinks
```

### `test_sites.py`
Build & integrity tests for the published output: the portal assembles, structure and
assets exist, every hub page carries the back-link, there are **no broken internal
links** across the whole site, and both vaults pass `verify_kb`. Zero dependencies;
also pytest-compatible.

```
python3 test_sites.py     # build + validate, exits 0/1  (or: pytest test_sites.py)
```

New knowledge bases are generated with the `knowledge-base-builder` Claude Code
skill, which builds, verifies, and publishes a KB here for any topic.
